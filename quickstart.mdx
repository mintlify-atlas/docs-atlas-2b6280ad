<Steps>
  <Step title="Clone the repository">
    Clone the skill repo to your local machine:

    ```bash
    git clone https://github.com/stat-guy/imessage-mail.git
    ```
  </Step>

  <Step title="Copy skill files to Claude Code">
    Create the skills directory and copy the skill files into it:

    ```bash
    mkdir -p ~/.claude/skills/imessage-mail
    cp -r imessage-mail/SKILL.md ~/.claude/skills/imessage-mail/
    cp -r imessage-mail/agents ~/.claude/skills/imessage-mail/
    ```
  </Step>

  <Step title="Grant Full Disk Access to your terminal">
    The skill reads SQLite databases that macOS protects behind Full Disk Access. You must grant access to whichever terminal you use with Claude Code.

    1. Open **System Settings** > **Privacy & Security** > **Full Disk Access**
    2. Click the **+** button
    3. Add your terminal app (Terminal.app, iTerm2, Warp, Ghostty, etc.)
    4. Restart the terminal

    <Note>
      If you skip this step, SQLite queries against `chat.db` and the Mail database will fail with a permissions error.
    </Note>
  </Step>

  <Step title="Allow AppleScript access (first send only)">
    The first time you send an iMessage or email, macOS will prompt you to allow your terminal to control **Messages.app** or **Mail.app** via AppleScript. Click **Allow**.

    This is a one-time prompt. If you accidentally deny it, re-enable it in **System Settings** > **Privacy & Security** > **Automation**.
  </Step>
</Steps>

## Using the skill

Once installed, invoke the skill in Claude Code using natural language:

```
/imessage-mail recent
/imessage-mail messages from Sarah
/imessage-mail search "project deadline"
/imessage-mail email
/imessage-mail send a text to Mike saying I'm running late
/imessage-mail find attachments from David
```
